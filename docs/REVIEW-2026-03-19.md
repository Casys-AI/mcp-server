# Architecture Review — 2026-03-19

Findings from automated architecture review of the compose/deploy flow.

## CRITICAL — Fixed

### 1. iframe sandbox breaks cross-origin postMessage
The event bus uses `sandbox="allow-scripts allow-same-origin"` but MCP UIs run on
different ports than the dashboard. Cross-origin postMessage is silently broken.
**Fix:** Proxy MCP UIs through the dashboard host port, or remove sandbox restriction.

### 2. composeDashboard() kills cluster before dashboard is served
The `finally` block in `composeDashboard()` stops the cluster, but iframes need
live HTTP endpoints. The URLs go dead.
**Fix:** Return cluster handle, let caller manage lifecycle. Add `keepAlive` option.

### 3. requiredEnv not validated at compose time
Missing credentials are only detected when the MCP process crashes. No pre-flight check.
**Fix:** Validate requiredEnv against available env vars before starting any server.

### 4. startAll() is sequential, no partial failure cleanup
One slow server blocks all others. If server #3 fails, servers #1-#2 are leaked.
**Fix:** Parallel startup with `Promise.allSettled`, cleanup on partial failure.

## IMPORTANT — To address next

### 5. postMessage("*") in deploy mode
Any iframe script can read all compose events including sharedContext.
**Action:** Thread actual target origins through postMessage dispatch.

### 6. No auth on shareable link
Anyone with the URL gets full data access (including local DB via tunnel).
**Action:** Add sessionToken generated at deploy time, verified by relay.

### 7. No tunnel reconnection
Laptop sleep → WebSocket drops → dashboard dead, no recovery.
**Action:** Auto-reconnect with backoff + relay grace period for session hold.

### 8. Sync rules not validated in validateTemplate()
Agent can generate invalid wiring (wrong tool names, missing emits/accepts).
**Action:** Cross-reference sync rules against manifest emits/accepts.

### 9. detectListenUrl timeout race
Reader lock leaked + stderr not drained on timeout path.
**Action:** Use AbortController, drain stderr in both paths.

### 10. windowToSlot map stale after navigation
Built once at DOM-ready, breaks after tab switches.
**Action:** Lazy lookup on each message event.

## MINOR — Track for later

### 11. {{placeholder}} not validated
Unfulfilled placeholders pass literal string to tool call.
**Action:** Pre-flight check that all placeholders in calls have matching args.

### 12. No generic database MCP
The "local DB" use case has no MCP implementation. Need at least a postgres stub.
**Action:** Create a stub-postgres MCP or document how to build one.

### 13. "Any app" claim overstated
No support for stateful UIs (pagination, forms, real-time push).
**Action:** Document limitations clearly. Consider polling/refresh mechanism.

### 14. No security headers in serveDashboard
No CSP, no X-Frame-Options. OK for local, risky for deploy.
**Action:** Add security headers in deploy mode.

### 15. Duplicate manifest names silently overwrite
Two files with same `name` in manifest directory → last wins, no error.
**Action:** Throw on duplicate names.
